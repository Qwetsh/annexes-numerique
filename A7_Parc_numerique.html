Parc numérique — outil de pilotage

Document interactif de présentation. Application web interne d'inventaire et de pilotage du parc informatique du collège, qui remplace le recensement statique par une vue dynamique de l'état de santé du matériel, salle par salle et étage par étage.

1 Le fil conducteur : un code d'états sémantiques

Une même grammaire de couleur parcourt toute l'interface, du badge jusqu'à la maquette 3D. L'état n'est jamais codé par la couleur seule : toujours doublé d'un libellé et d'une icône (accessibilité WCAG AA).

Vert · fonctionnel Ambre · vétuste Rouge · en panne Gris · réformé

3 Vue du collège — explorez un étage

La version en ligne propose une maquette 3D Three.js où chaque salle est teintée selon la santé de son matériel. Voici une version simplifiée et cliquable : choisissez un étage, puis une salle pour afficher son détail.

Sélectionnez une salle

Cliquez sur une salle de l'étage pour voir le détail de son matériel et de son état.

Le R+1 est l'étage « réel » de la démonstration (CDI, salle informatique 210, labo…) ; les autres étages illustrent le code couleur.

4 Les écrans de l'application

Tableau de bord

Indicateurs clés, répartition par état, parc par étage, salles à surveiller, répartition par type.

Inventaire des équipements

Table triable avec recherche et filtres (type, état, étage) + filtre rapide « + de 5 ans ».

Vue du collège (3D)

Maquette interactive : la salle s'élève et s'illumine au clic, le reste s'estompe, panneau latéral détaillé.

Système de design

Palette, états sémantiques, échelle typographique, rayons, ombres, composants réutilisables.

5 Technique & accès

Stack

Vite + React 18 + TypeScript React Router Three.js / react-three-fiber Design tokens CSS Prévu pour Supabase WCAG AA

Données de démonstration fictives ; l'outil est pensé pour être branché à une base de données réelle. Déploiement automatique sur GitHub Pages.
